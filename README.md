# OIer Toolkit

OIer 数据生成与对拍辅助工具（VS Code 扩展）。

## 功能

### 智能数据构造器

粘贴 OI 题目的输入格式描述，自动解析并生成随机测试数据。

**支持的格式：**

| 格式        | 示例                               |
| --------- | -------------------------------- |
| 单变量不等式    | `1 ≤ N ≤ 200`                    |
| 并列变量不等式   | `1 ≤ A, B ≤ N`                   |
| 数组声明（中文）  | `N 个非负整数，表示 K_i（0 ≤ K_i ≤ N）`    |
| 数组声明（行输入） | `接下来 N 行，每行一个整数 a_i ∈ [1, 10^9]` |
| 无不等式的变量声明 | `第一行两个整数 N, M`                   |
| 元组/边集输入   | `接下来 M 行，每行三个整数 u, v, w`         |
| 二维矩阵      | `一个 N × M 的矩阵`                   |
| 科学计数法     | `2 × 10^5`，`-10^3`               |
| T 组测试数据   | `第一行一个整数 T，表示有 T 组测试数据`          |
| 百分比数据范围   | `对于 40% 的数据，N ≤ 1000`            |

### 两种生成模式

#### 简单模式（Simple）

粘贴 → 解析 → 设置组数 → 生成随机数据。适合快速生成少量测试点。

#### 子任务模式（Subtask）

粘贴 → 解析 → 选择预设模板 → 一键生成按百分比分布的完整测试数据。

**内置 3 套预设模板：**

- **基础百分比分布**：10% 小数据 + 30% 中数据 + 60% 较大数据 + 100% 全数据
- **边界强化分布**：全最小值 + 全最大值 + 小数据 + 全数据
- **极端测试分布**：全最小 + 全最大 + 混合边界 + 50% + 100% 随机

**策略体系：**

| 策略                    | 功能         | 示例                      |
| --------------------- | ---------- | ----------------------- |
| ScaleStrategy         | 按百分比缩放变量范围 | 10% 时 q 取 `[3, 105]`    |
| BoundaryValueStrategy | 生成边界值      | allMin / allMax / mixed |
| RandomStrategy        | 范围内随机生成    | 默认随机                    |
| SpecialStrategy       | 预留特殊性质接口   | 待扩展                     |

### 百分比约束解析

解析题面中的 `对于 X% 的数据` 块，在简单模式和子任务模式中均可选择不同的数据规模约束。

### 快捷输入模板

内置 9 个常见 OI 输入格式模板，一键填入：

- 数组 + 操作 / 图论边集 / 带权边 / 矩阵 / 百分比约束 / T 组数据 / 序列操作 / 树结构 / 含百分比矩阵

**自定义模板：** 点击 **+ 自定义** 将当前输入保存为自定义模板，持久化存储在本地。

### 数据保存

| 保存方式        | 行为                                           |
| ----------- | -------------------------------------------- |
| 单组保存        | 弹对话框保存单个 `.in` 文件                            |
| 保存全部（简单模式）  | 全部保存到 `testcases/test1.in`、`test2.in`……      |
| 保存全部（子任务模式） | 按子任务分文件夹保存 + 自动生成 `config.yml` + `README.md` |

**子任务模式保存结构：**

```
testcases/
├── 全最小值/
│   ├── test1.in
│   └── test2.in
├── 全最大值/
│   ├── test3.in
│   └── test4.in
├── 30%_小数据/
│   ├── test5.in
│   ├── test6.in
│   └── test7.in
├── 50%_全数据/
│   ├── test8.in
│   ├── test9.in
│   └── test10.in
├── config.yml      ← 洛谷格式子任务配置
└── README.md       ← 测试数据说明文档
```

## 使用方法

### 简单模式

1. 在侧边栏（活动栏中的烧杯图标）找到 OIer Toolkit
2. 点击顶部模板按钮快速填入，或在文本框中粘贴题目输入格式描述
3. 点击 **解析并预览** 查看解析结果
4. 保持 **简单模式**，设置生成组数，点击 **生成数据**
5. 点击 **保存** 保存单组，或 **保存全部** 批量保存

### 子任务模式

1. 粘贴并解析题目描述
2. 点击 **子任务模式** 切换
3. 选择一个预设模板
4. 点击 **生成数据** — 自动按子任务生成全部测试点
5. 点击 **保存全部** — 按子任务分文件夹保存 + config.yml + README

### 示例输入

```
第一行为三个用空格隔开的正整数，表示 N,A,B（1≤N≤200，1≤A,B≤N）。
第二行为 N 个用空格隔开的非负整数，表示 K_i（0≤K_i≤N）。
```

```
第一行两个整数 N, M，1 ≤ N ≤ 50000，1 ≤ M ≤ 200000
接下来 M 行，每行三个整数 u, v, w，1 ≤ u, v ≤ N，-10^3 ≤ w ≤ 10^3
```

```
1 ≤ N, M ≤ 50
接下来一个 N × M 的矩阵
```

```
第一行一个整数 T，表示有 T 组测试数据（1≤T≤10）。
每组第一行两个整数 N, M（1≤N≤1000，1≤M≤N）。
每组第二行 N 个整数 a_i（1≤a_i≤10^9）。
```

```
第一行一个正整数 N（1 ≤ N ≤ 10^5）。
第二行 N 个整数 a_i（1 ≤ a_i ≤ 10^9）。

对于 30% 的数据，N ≤ 1000。
对于 60% 的数据，N ≤ 50000。
对于 100% 的数据，N ≤ 10^5。
```

## 架构

```
粘贴题面文本
    ↓
parseNaturalLanguage()          ← 自然语言解析器
    ↓                                    （变量 + 数组 + 边集 + T 组 + 百分比）
ParseResult
    ↓
┌─ simple: generateData()       ← 两阶段随机生成
└─ subtask: executeSubtask()    ← 策略链驱动
       ↓                              scale / boundary / random
   子任务数据
    ↓
testcases/                      ← 按文件夹保存 + config.yml + README
```

### 核心模块

| 文件                                    | 说明                          |
| ------------------------------------- | --------------------------- |
| `src/core/parser/naturalParser.ts`    | 自然语言解析器，将题面文本转为结构化配置        |
| `src/core/parser/types.ts`            | 类型定义（含策略、子任务、GenOptions）    |
| `src/core/generator/dataGenerator.ts` | 生成器主入口 + 子任务执行引擎            |
| `src/core/generator/strategies.ts`    | 策略实现（Scale、Boundary、Random） |
| `src/core/generator/templates.ts`     | 3 套预设子任务模板                  |
| `src/core/generator/configWriter.ts`  | config.yml + README 自动生成    |
| `src/core/generator/genUtils.ts`      | 共享工具函数                      |
| `src/webview/DataGenPage.tsx`         | 前端交互界面                      |
| `src/webview/inputTemplates.ts`       | 快捷输入模板数据                    |
| `src/extension.ts`                    | VS Code 扩展入口，处理文件保存等消息      |

## 开发

```bash
# 安装依赖
npm install

# 构建前端（webview）
npx vite build --config src/webview/vite.config.mts

# 构建后端（extension）
npm run compile

# 启动调试
F5  # 在 VS Code 中按 F5
```
